# Daily Planner Web: Google広告/アフィリエイト収益化フレームワーク

このドキュメントは、月額課金の前に `Google系の広告収益` を先行実装するための実行枠組みです。

## 0. 前提整理

- `Google Ads` は広告出稿（お金を使う側）の仕組み
- まず収益化したい場合は `Google AdSense`（広告掲載で収益化）が主軸
- ここでは「広告収益 + 必要に応じて外部アフィリエイト導線」のハイブリッドを対象にする

## 1. 収益化の基本方針

- 方針: UX を壊さない範囲で広告収益を作る
- 優先順位:
1. 継続利用率を落とさない
2. 広告 CTR / RPM を改善する
3. 課金機能（将来のサブスク）へ接続できるデータを残す

## 2. 配置ルール（最初の実装）

初期は「少数・高意図」だけに絞る。

- 画面A: ダッシュボード下部に 1 枠
- 画面B: 設定画面の下部に 1 枠
- 画面C: 週次振り返りエリアに 1 枠

禁止:
- タスク入力フォームの近く
- 主要CTAボタン周辺
- 画面上部固定で常時表示

## 3. KPI 設計

毎週見る指標を固定する。

- Product KPI:
  - DAU / WAU
  - D1, D7 継続率
  - 主要操作完了率（例: タスク作成率）
- Revenue KPI:
  - 広告表示回数（Impressions）
  - CTR
  - RPM（1000表示あたり収益）
  - 1ユーザーあたり広告収益（ARPDAU）
- Guardrail KPI:
  - 離脱率の急増
  - ページ表示速度悪化
  - 苦情/ブロック率

## 4. イベント計測（最小）

- `ad_slot_rendered`（slot_id, page, date）
- `ad_slot_clicked`（slot_id, page, date）
- `ad_slot_blocked_or_empty`（slot_id, reason）
- `task_created`（既存の価値指標）

目的:
- 広告成績だけでなく、プロダクト価値を毀損していないかを同時に判定する

## 5. リリースフェーズ

### Phase 1（1〜2週間）

- [ ] 2〜3枠だけ実装
- [ ] 週次で KPI モニタリング

### Phase 2（3〜4週間）
- [ ] 低成果枠の停止

### Phase 3（5週目以降）
- [ ] 収益寄与の高い面に限定再配置
- [ ] サブスク導線と比較（広告表示あり/なし）
- [ ] サブスク化判断に必要なデータを確定

## 6. 意思決定ルール（継続/停止）

- 継続条件:
  - 継続率が悪化していない
  - RPM が一定水準を超える
  - 主要導線の完了率が維持される
- 停止条件:
  - D7 継続率が明確に悪化
  - UX苦情が継続発生
  - 収益が運用コストに見合わない

## 7. Cloudflare運用上の実装メモ

- `feature/*` ブランチで広告枠を実装し、PR Preview で表示確認
- 本番は `main` マージ時のみ反映
- CSP設定で広告スクリプトの許可ドメインを管理
- パフォーマンス監視（LCP / CLS）を広告導入前後で比較
